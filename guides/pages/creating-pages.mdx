## Overview

Pages in Radical Whale come in two types: workspace-level pages for general documentation and record-level pages for specific dataset entries. Both offer rich editing capabilities powered by the Tiptap editor with AI integration.

## Creating Workspace Pages

Workspace pages are accessible to all members of your workspace and serve as your team's documentation hub.

### Step-by-Step Creation

1. **Navigate to Pages**

   - Click "Pages" in your workspace sidebar
   - This opens the pages list view

2. **Start New Page**

   - Click the "Create Page" button
   - A new page editor opens immediately

3. **Add Title and Icon**

   - Enter a descriptive title in the title field
   - Click the icon/emoji area to select a visual identifier
   - Choose from emoji picker or Lucide icon library

4. **Set Initial Status**

   - Select "Draft" for work-in-progress content
   - Choose "Published" when ready to share
   - Use "Archived" for historical reference

5. **Add Tags** (Optional)

   - Click "Add tags" to categorize your page
   - Type tag names or select from existing tags
   - Tags help with organization and discovery

6. **Start Writing**
   - Click in the editor area to begin
   - Content auto-saves as you type
   - Use markdown or formatting toolbar

## Creating Record Pages

Record pages are attached to specific dataset records and provide contextual documentation for individual data entries.

### From Dataset Record View

1. **Open Record Details**

   - Navigate to any dataset
   - Click on a record to open its detail view

2. **Access Pages Tab**

   - Click the "Pages" tab in the record detail panel
   - See any existing pages for this record

3. **Create New Page**

   - Click "Create Page" button
   - Page is automatically linked to the record

4. **Configure Page**

   - Add title and icon as with workspace pages
   - Page inherits record context automatically
   - Start writing with full access to record data

5. **Use Record Context**
   - AI has access to all record field values
   - Reference other columns in your content
   - Generate content informed by record data

## Best Practices

### Naming Conventions

**Dates in Titles**

Include dates when relevant:

- "Q4 2024 Board Report"
- "Weekly Update - 2024-10-21"
- "Market Research - October 2024"

**Project Prefixes**

Use consistent prefixes:

- "[Project Alpha] Technical Design"
- "[Marketing] Campaign Brief"
- "[Sales] Territory Analysis"

**Version Indicators**

Show version in title when needed:

- "API Documentation v2.0"
- "Style Guide - Updated 2024"
- "Process Doc (Latest)"

### Content Organization

**Start with Structure**

Begin with clear headings:

```markdown
# Main Title

## Section 1

### Subsection 1.1

### Subsection 1.2

## Section 2

### Subsection 2.1
```

**Use Lists Effectively**

- Bullet points for unordered items
- Numbered lists for sequences
- Task lists for action items
- Nested lists for hierarchy

**Include Visual Breaks**

Use horizontal rules to separate major sections:

```markdown
## Section 1

Content here...

---

## Section 2

Content here...
```

### Collaboration Guidelines

**Clear Ownership**

- Include author name or team
- Specify point of contact
- Note last updated date
- Indicate review status

**Regular Updates**

- Update content as information changes
- Add dates to major revisions
- Archive outdated pages
- Link to newer versions

**Accessible Writing**

- Use clear, simple language
- Define acronyms and jargon
- Include context and background
- Provide links to related pages

## Common Use Cases

### 1. Team Documentation

**Purpose**: Create shared knowledge base

**Setup**:

- Create workspace pages for different topics
- Use consistent naming and tagging
- Organize with hierarchical structure
- Link related pages together

**Maintenance**:

- Regular review cycles
- Update as processes change
- Archive old procedures
- Onboard new members with documentation

### 2. Project Management

**Purpose**: Track project information and progress

**Setup**:

- Create project hub page with links
- Sub-pages for different aspects
- Use task lists for action items
- Regular status updates

**Updates**:

- Weekly progress updates
- Link to related resources
- Track decisions and changes
- Document lessons learned

### 3. Research and Analysis

**Purpose**: Document research findings

**Setup**:

- Create page for each research initiative
- Use record pages for specific cases
- Link to data sources
- Include methodology notes

**Structure**:

- Clear research questions
- Methodology documentation
- Findings and evidence
- Insights and recommendations

### 4. Customer Documentation

**Purpose**: Track customer interactions and information

**Setup**:

- Create record pages for customer records
- Document conversations and notes
- Track preferences and history
- Link to related datasets

**Usage**:

- Add notes after each interaction
- Update contact information
- Track purchase history
- Document support issues
