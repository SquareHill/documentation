## Overview

Datasets are the foundation of data processing in Radical Whale. This guide will walk you through creating datasets, defining their structure, and preparing them for AI agent processing.

## Prerequisites

- Access to a Radical Whale workspace
- Basic understanding of your data structure
- Sample data file (CSV recommended) or plan for manual data entry

## Step 1: Navigate to Datasets

### Accessing the Datasets Section

1. Log into your Radical Whale workspace
2. Click **Datasets** in the left sidebar
3. You'll see the datasets overview page showing existing datasets (if any)

[Screenshot Placeholder: Datasets overview page]

### Understanding the Datasets Interface

- **Datasets List**: Shows all datasets in your workspace
- **Create Button**: Primary action to create new datasets
- **Search & Filter**: Find specific datasets quickly
- **Import Options**: Different ways to add data

## Step 2: Create a New Dataset

### Starting Dataset Creation

1. Click the **Create Dataset** button
2. You'll be presented with the dataset creation form

### Basic Information

#### Dataset Name

Choose a descriptive name that clearly identifies your data:

- **Good examples**: "Customer Contact Data", "Product Inventory Q4", "Survey Responses 2024"
- **Avoid**: Generic names like "Dataset 1" or unclear abbreviations

#### Description

Add a detailed description explaining:

- What type of data this dataset contains
- The source of the data
- How it will be used
- Any important context or notes

Example:

```
Customer contact information collected from our website sign-up forms
between January-March 2024. Includes company names, contact details,
and initial interest areas. Will be used for lead qualification and
sales outreach preparation.
```

[Screenshot Placeholder: Dataset creation form]

<Tip>
  Detailed descriptions help team members understand the dataset's purpose and
  context, making collaboration more effective.
</Tip>

## Step 3: Define Dataset Structure

### Column Planning

Before creating the dataset, plan your column structure:

#### Essential Columns

- **Unique Identifier**: A column that uniquely identifies each record
- **Primary Data**: The main information you want to process
- **Context Columns**: Additional information that provides context
- **Output Columns**: Where processed results will be stored

#### Column Types to Consider

- **Text Fields**: Names, descriptions, addresses
- **Contact Information**: Emails, phone numbers, websites
- **Numeric Data**: Quantities, scores, financial amounts
- **Dates**: Creation dates, deadlines, events
- **Categories**: Industry, status, priority levels

### Initial Column Setup

You can either:

1. **Start Empty**: Create the dataset structure first, then add data
2. **Import with Data**: Create columns based on imported CSV headers
3. **Template Approach**: Use a predefined template for common use cases

[Screenshot Placeholder: Column planning interface]

## Step 4: Configure Dataset Settings

### Access Control

- **Visibility**: Who can see this dataset within the workspace
- **Edit Permissions**: Who can modify the data and structure
- **Processing Permissions**: Who can run AI agents on this data

### Data Quality Settings

- **Validation Rules**: Set up automatic data validation
- **Required Fields**: Specify which columns must have values
- **Format Constraints**: Define acceptable formats for specific columns

### Processing Configuration

- **Batch Size**: How many records to process at once
- **Timeout Settings**: Maximum time for processing operations
- **Error Handling**: How to handle processing failures

[Screenshot Placeholder: Dataset settings panel]

## Step 5: Create the Dataset

### Final Review

Before creating, review:

- Dataset name and description
- Planned column structure
- Access and permission settings
- Processing configuration

### Create Dataset

1. Click **Create Dataset** to finalize
2. The system will create your dataset structure
3. You'll be redirected to the dataset detail page

### Confirmation

You should see:

- Dataset successfully created message
- Empty dataset with your planned structure
- Options to add columns or import data

[Screenshot Placeholder: Newly created dataset page]

## Step 6: Add Columns (If Starting Empty)

### Column Creation Process

1. Click **Add Column** on the dataset page
2. Configure each column with the following details:

#### Basic Column Settings

- **Column Name**: Display name (e.g., "Company Name")
- **Column Key**: System identifier (e.g., "company_name")
- **Data Type**: Select from available types
- **Description**: Explain what this column contains

#### Data Types Available

- **Short Text**: Names, titles, brief descriptions (up to 255 chars)
- **Long Text**: Detailed descriptions, content, notes
- **Email**: Email addresses with validation
- **URL**: Website addresses with validation
- **Phone**: Phone numbers with formatting
- **Number**: Numeric values (integer or decimal)
- **Date**: Date values with various formats
- **Boolean**: True/false values
- **JSON**: Complex structured data

#### Processing Configuration

- **AI Instructions**: How agents should process this column
- **Agent Assignment**: Which agent should process this column (set later)
- **Dependencies**: Which other columns this depends on
- **Validation Rules**: Data quality requirements

[Screenshot Placeholder: Column creation form]

### Example Column Configuration

#### Company Name Column

```
Name: Company Name
Key: company_name
Type: Short Text
Description: Official company name from website or business registration
Instructions: Extract the official company name, removing any suffixes like LLC, Inc, etc. unless they are part of the brand identity
Required: Yes
Validation: Minimum 2 characters, no special characters except spaces and hyphens
```

#### Contact Email Column

```
Name: Contact Email
Key: contact_email
Type: Email
Description: Primary contact email address for the company
Instructions: Find the main contact email, preferably a specific person rather than info@ or support@
Required: No
Validation: Valid email format, no generic emails like noreply@
```

## Step 7: Import Initial Data (Optional)

### CSV Import Process

If you have existing data to import:

1. Click **Import Data** or **Import CSV**
2. Upload your CSV file
3. Map CSV columns to dataset columns
4. Review data preview
5. Configure import settings
6. Execute import

### Import Configuration

- **Header Row**: Specify if first row contains headers
- **Delimiter**: Comma, semicolon, tab, or custom
- **Encoding**: UTF-8, ASCII, or other character encodings
- **Quote Character**: How text fields are quoted in the CSV

### Column Mapping

- Match CSV headers to your dataset columns
- Create new columns for unmapped CSV fields
- Skip columns you don't need
- Transform data during import if needed

## Step 8: Validate Dataset Structure

### Data Quality Check

After creation and initial data import:

1. **Review Columns**: Ensure all necessary columns are present
2. **Check Data Types**: Verify data types match your data
3. **Test Validation**: Try adding sample data to test validation rules
4. **Review Relationships**: Confirm column dependencies are correct

### Common Validation Issues

- **Data Type Mismatches**: Numbers stored as text, dates in wrong format
- **Missing Required Fields**: Empty values in required columns
- **Format Problems**: Invalid emails, URLs, or phone numbers
- **Encoding Issues**: Special characters not displaying correctly

### Fixing Issues

- **Edit Columns**: Modify column definitions as needed
- **Data Transformation**: Clean up data formatting issues
- **Re-import**: Fix source data and re-import if necessary
- **Manual Correction**: Fix individual records manually

## Best Practices

### Dataset Design

- **Keep It Focused**: Each dataset should serve a specific purpose
- **Consistent Naming**: Use clear, consistent naming conventions
- **Document Everything**: Add descriptions to datasets and columns
- **Plan for Growth**: Design structure that can accommodate future needs

### Column Organization

- **Logical Grouping**: Group related columns together
- **Clear Dependencies**: Make column relationships explicit
- **Avoid Redundancy**: Don't duplicate information unnecessarily
- **Consider Processing**: Design with AI agent processing in mind

### Data Quality

- **Validation Rules**: Set up appropriate validation for each column
- **Required Fields**: Mark essential columns as required
- **Format Standards**: Establish consistent formatting rules
- **Regular Reviews**: Periodically review and clean data

### Security Considerations

- **Sensitive Data**: Be careful with personally identifiable information
- **Access Control**: Set appropriate permissions for different team members
- **Data Retention**: Plan for how long data should be kept
- **Compliance**: Ensure compliance with relevant data protection regulations

## Troubleshooting

### Common Issues

**Dataset Creation Fails**

- Check workspace permissions
- Verify all required fields are filled
- Ensure dataset name doesn't conflict with existing datasets
- Try refreshing the page and starting over

**Column Configuration Problems**

- Review data type selections for your data
- Check validation rules aren't too restrictive
- Ensure column names are unique
- Verify required field settings are appropriate

**Import Issues**

- Check CSV file formatting and encoding
- Verify column mapping is correct
- Look for special characters or formatting issues
- Try importing a smaller sample first

**Performance Problems**

- Large datasets may take time to process
- Consider breaking large datasets into smaller chunks
- Review column validation complexity
- Contact support for optimization advice
